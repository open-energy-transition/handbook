# Technology stack that we use

Recommended, not enforced.
Projects may deviate, especially in case of projects based on soft forks.

* Documentation: `mkdocs-material`
* Code: `python`
* Pre-commit framework: `pre-commit`
    * Public repositories: use `pre-commit.ci` for automatic checks
    * Example file: TODO link
* Code style: `black` via `ruff`
* Code linting: `ruff`
* Licensing and copyright:
    * Adding: `reuse`
    * Checking: `reuse` via `pre-commit`
